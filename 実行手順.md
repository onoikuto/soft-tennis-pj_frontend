# アプリの実行手順

## 1. ターミナルを開く

プロジェクトディレクトリに移動します：
```bash
cd /Users/ononono/soft-tennis-pj_frontend
```

## 2. PATHの設定（初回のみ）

Flutterのパスを設定します：
```bash
export PATH="$PATH:/Users/ononono/soft-tennis-pj_frontend/flutter/bin"
```

または、`~/.zshrc`に既に追加済みの場合は、新しいターミナルを開くか：
```bash
source ~/.zshrc
```

## 3. 依存関係のインストール

```bash
flutter pub get
```

## 4. 利用可能なデバイスの確認

```bash
flutter devices
```

以下のような出力が表示されます：
- iOS Simulator（Macの場合）
- Android Emulator（Android Studioがインストールされている場合）
- 接続された実機デバイス

## 5. アプリの実行

### 方法1: 自動的にデバイスを選択
```bash
flutter run
```

### 方法2: 特定のデバイスを指定
```bash
flutter run -d <device_id>
```

例：
```bash
# iOS Simulatorの場合
flutter run -d iPhone

# Android Emulatorの場合
flutter run -d emulator-5554
```

## 6. ホットリロード

アプリが起動したら、コードを変更すると自動的にリロードされます：
- `r` キー: ホットリロード
- `R` キー: ホットリスタート
- `q` キー: アプリを終了

## ⚠️ 初回実行時の注意点

**`flutter devices`や`flutter run`の初回実行には時間がかかります（5-10分程度）**

理由：
1. **Dart SDKのダウンロード**: 約200MBのDart SDKをダウンロード（初回のみ）
2. **Flutterツールのビルド**: Flutterコマンドラインツールをコンパイル
3. **デバイスの検出**: iOS SimulatorやAndroid Emulatorを検出
4. **依存関係の解決**: プロジェクトの依存パッケージを分析

**進捗の確認方法**:
- ターミナルに「Downloading...」「Building...」などのメッセージが表示されます
- ネットワーク接続が必要です
- 初回以降はキャッシュが使われるため、より高速になります

## トラブルシューティング

### Flutterが見つからない場合
```bash
which flutter
```
パスが表示されない場合は、PATHを再設定してください。

### デバイスが見つからない場合
- iOS: Xcodeをインストールし、Simulatorを起動
- Android: Android Studioをインストールし、エミュレータを起動

### ビルドエラーが発生する場合
```bash
flutter clean
flutter pub get
flutter run
```

### コマンドが長時間かかる場合
- 初回実行時は正常です（5-10分かかることもあります）
- ネットワーク接続を確認してください
- キャッシュをクリアして再試行：
  ```bash
  rm -rf flutter/bin/cache
  flutter doctor
  ```
